General

How do I log in?

Users will be required to login to the web app before they are allowed access to users, projects, or customers databases. The login credentials include "email" and "password". Users can also select the option "Forgot your password?" if they forget their log in credentials. Users are given a "Remember Me" option that will keep them signed in if they access the application using the same machine. Once users successfully log in, the web application will redirect to the main projects page.

Projects

Navigating through the All Projects page :

The All Project page shows a table that lists all of the ongoing projects that are saved on the database. The column titles include: Name, Type, Customer, Stage, Sales Delivery, Rental Delivery, and Rental Pickup. Each project has options of Show, Edit, and Destroy.

Show Option : provides the users with a more detailed information page for the selected project. Additional information includes:

Attachments : Any document related to the selected project

Customer Interactions : Past customer interactions displayed in table form

Past Projects : Past projects involving the same customer

Edit Option : allows the users to edit project details including: Name, Project Type, Customer, Project Stage, Sales Delivery, Rental Delivery, Rental Pickup, and Attachments

Destroy Option : removes the selected project from the database


Adding a new project :

Users can navigate to the New Project tab. The page will prompt users for information required to create a new project. Required information: Name, Type, Stage, Customer, Sales Delivery, Rental Delivery, and Rental Pickup. When Create Project is selected, the web application will validate the information entered by the user and save the project if information is valid or output error messages if information is invalid.

Customers

Navigating through the All Customers page :

The All Customers page shows a table that lists all customers that are saved on the database. The columns include: Name, Company, Email, Phone, Cell, Address, Created At, and Updated At. Each Project has the options of Show, Edit, and Delete.

Show Option : provides the users with a more detailed information page for the selected customer. Additional information includes:

Customer Interactions : Past customer interactions recorded in table form

Past Projects : Past projects involving the same customer

Edit Option : allows the users to edit customer details including: Name, Company, Email, Phone, Cell, and Address.

Destroy Option : removes the selected customer from the database


Adding a new customer :

Users can navigate to the New Customer tab. The page will prompt users for information required to create a new customer. Required information: Name, Company, Email, Phone, Cell, and Address. When Create Customer is selected, the web application will validate the information entered by the user and save the customer if information is valid or output error messages if information is invalid.

Inventory

Navigating through the All Inventory page :

The All Inventory page shows a table that lists all inventory that are saved on the database. The columns include: Name, Manufacturer, Model Number, and Color. Each inventory has the options of Show, Edit, and Delete.

Show Option : provides the users with a more detailed information page for the selected inventory style.

Edit Option : allows the user to edit inventory information including: Name, Manufacturer, Model Number, and Color.

Destroy Option : removes the inventory style from the database


Adding a new inventory :

Users can navigate to the New Inventory tab. The page will prompt users for information required to create a new inventory style. Required information: Name, Manufacturer, Model Number, and Color. Users also have the option to attach files related to the inventory. When Save is selected, the web application will validate the information entered by the user and save the inventory if information is valid or output error messages if information is invalid.

Users

Navigating through the All Users page :

The All Users page shows a table that lists all users that are saved on the database. The columns include: Name, Email, and Role. Each inventory has the options of Show, Edit, and Delete.

Show Option : provides the user with a more detailed information page for the selected users. Additional information includes: User ID

Edit Option : allows the user to edit user information including: Name, Password, Email, Role (Sales, Customer Service, Warehouse, Design).

Destroy Option : removes the selected user from the database


Adding a new user :

Users can navigate to the New User tab. The page will prompt users for information required to create a new user. Required information: Name, Password, Password Confirmation, and Email. Users also have the option to add the Role of the new user. When Create User is selected, the application will validate the information entered by the user and save the style if all required information is valid or output error messages if information is invalid.
